## 🌐 t2tunnel چیست؟

`t2tunnel` یک تونلِ سبک، سریع و چندلایه است که با **Go** نوشته شده و یک کار را عالی انجام می‌دهد:

> **رساندنِ ترافیک از پشتِ سانسور و فیلترینگ به اینترنتِ آزاد — بدونِ اینکه DPI بفهمد چه خبر است.**

t2tunnel ترافیک را در چند لایه می‌پیچد و آن را شبیهِ یک اتصالِ کاملاً عادی نشان می‌دهد. به‌جای اینکه با ده‌ها دستورِ پیچیده‌ی ترمینال کلنجار بروی، **همه‌چیز از یک پنلِ تحتِ وب** مدیریت می‌شود: نصب، ساختِ کانفیگ، استارت، لاگ، همه با چند کلیک.

این ابزار معمولاً به‌صورتِ **رله** کار می‌کند: کاربر به یک سرورِ داخلیِ تمیز وصل می‌شود، آن سرور ترافیک را از طریقِ تونل به سرورِ خارج می‌رساند، و سرورِ خارج آن را به اینترنتِ آزاد تحویل می‌دهد.

## ✨ قابلیت‌های تونل (همه از داخلِ پنل)

پنل به تو اجازه می‌دهد بینِ چند نوع تونل انتخاب کنی، هرکدام برای یک شرایط:

| | لایه | شبیهِ چه می‌شود | کاربرد |
|:---:|---|---|---|
| ⚡ | **KCP / UDP** | ترافیکِ UDP پرسرعت | سرعتِ بالا، تأخیرِ کم |
| 🎭 | **TLS / HTTPS Mimicry** | یک سایتِ HTTPS عادی | عبور از DPI به‌صورتِ نامحسوس |
| 🌊 | **WebSocket** | یک اتصالِ WebSocket | آماده‌ی عبور از CDN |
| 🥷 | **Raw / QuantumMux** | بسته‌ی TCP خام | دور زدنِ عمیق (تجربی) |

و زیرِ همه‌ی این‌ها:

🔗 **smux** (چند استریم روی یک اتصال) · 🔐 **AES-GCM** · 🔁 **اتصالِ خودکارِ مجدد** · 🔀 **مبهم‌سازیِ ترافیک** · 💓 **Heartbeat** · 🛣️ **Multi-Path Failover** (چند سرور؛ یکی افتاد، می‌رود بعدی)

> 💡 برای استفاده‌ی روزمره، حالتِ **TLS** (همان HTTPS Mimicry) توصیه می‌شود — پایدار، نامحسوس و بدونِ نیاز به sudo.

## 📦 نصبِ پنل

روی **سرور** (چه ایران، چه خارج) این مراحل را بزن:

```bash
# ۱. پروژه را بگیر
git clone https://github.com/t2hash/t2tunnel.git
cd t2tunnel

# ۲. اسکریپتِ نصب را اجرا کن
sudo bash install.sh
```

اسکریپت خودش:
- پیش‌نیازها (Go، libpcap، build-essential) را نصب می‌کند
- پنل را build می‌کند
- یک **رمزِ تصادفی** می‌سازد
- پنل را روی `127.0.0.1:2331` بالا می‌آورد
- لینکِ ورود و رمز را در ترمینال چاپ می‌کند

خروجی چیزی شبیهِ این است:

```
PANEL IS UP — finish setup in your browser
URL        http://127.0.0.1:2331
username   admin
password   a3f9c2e1b8d4...      ← این رمز را نگه‌دار
```

> 🔑 رمز فقط یک‌بار نمایش داده می‌شود. اگر گمش کردی، فایلِ `panel.db.json` را پاک کن و دوباره نصب را اجرا کن.

## 💻 وارد شدن به پنل از سیستمِ شخصی

پنل برای امنیت فقط روی `127.0.0.1` (localhost) بالا می‌آید و **از بیرون مستقیم در دسترس نیست**. برای باز کردنش از کامپیوترِ خودت، یک **تونلِ SSH** می‌زنی.

روی **ویندوزِ خودت** (PowerShell) یا ترمینالِ لینوکس/مک:

```bash
ssh -L 2331:127.0.0.1:2331 root@SERVER_IP
```

سپس در مرورگر باز کن:

```
http://127.0.0.1:2331
```

با `admin` و رمزی که هنگامِ نصب گرفتی وارد شو. ✅

> 🧠 این دستور می‌گوید: «پورتِ ۲۳۳۱ کامپیوترِ من را از پشتِ همین تونلِ امنِ SSH به پورتِ ۲۳۳۱ سرور وصل کن.» تا وقتی این پنجره باز است، پنل در دسترس است.

## 🌍 نصبِ پنل روی سرورِ خارج (با پورتِ متفاوت)

اگر می‌خواهی **همزمان** هم پنلِ سرور ایران و هم پنلِ سرور خارج را روی کامپیوترت باز داشته باشی، باید برای سرورِ خارج یک **پورتِ متفاوت** استفاده کنی تا با پنلِ ایران تداخل نکند.

**روی سرورِ خارج، پنل را روی پورتِ ۲۳۳۲ بالا بیار:**

```bash
cd ~/t2tunnel/webpanel
go build -o t2panel .
sudo ./t2panel -addr 127.0.0.1:2332 -repo ~/t2tunnel
```

**سپس روی کامپیوترت، یک تونلِ SSH دوم با همان پورتِ ۲۳۳۲ بزن** (پنجره‌ی پنلِ ایران را نبند):

```bash
ssh -L 2332:127.0.0.1:2332 root@FOREIGN_IP
```

حالا دو تب در مرورگر داری:

| پنل | آدرس |
|---|---|
| 🇮🇷 سرور ایران | `http://127.0.0.1:2331` |
| 🌍 سرور خارج | `http://127.0.0.1:2332` |

### چرا پورتِ متفاوت؟

`127.0.0.1` روی هر دستگاه یعنی «خودِ همان دستگاه». پنلِ هر سرور روی `127.0.0.1:2331` **خودش** بالاست و تداخلی با سرورِ دیگر ندارد (چون دو ماشینِ جدا هستند).

تداخل **روی کامپیوترِ تو** اتفاق می‌افتد: یک کامپیوتر نمی‌تواند پورتِ ۲۳۳۱اش را همزمان به دو سرور وصل کند. برای همین پورتِ سمتِ چپِ `ssh -L` باید فرق کند (۲۳۳۱ برای ایران، ۲۳۳۲ برای خارج). برای سادگی، پنلِ سرورِ خارج را هم مستقیماً روی ۲۳۳۲ بالا می‌آوریم تا همه‌چیز یک‌دست بماند.

## ☁️ قابلیتِ استفاده از CDN

t2tunnel می‌تواند از پشتِ **CDN (مثل Cloudflare یا ArvanCloud)** رد شود. این یعنی:

- آی‌پیِ سرورِ خارج **مخفی** می‌ماند
- حتی اگر آی‌پیِ سرور فیلتر شود، چون کاربر به آی‌پیِ CDN وصل می‌شود، **بازم کار می‌کند**
- سرتیفیکیتی که DPI می‌بیند، **سرتِ واقعیِ CDN** است — یعنی اتصال شبیهِ یک HTTPSِ کاملاً واقعی به نظر می‌رسد

برای این کار از حالتِ **WebSocket** استفاده می‌شود (چون CDNها فقط WebSocket/HTTP را پروکسی می‌کنند، نه TLS خام را). یک دامنه روی Cloudflare اضافه می‌کنی، پروکسی (ابرِ نارنجی) را روشن می‌کنی، SSL را روی **Flexible** می‌گذاری، و تونلِ ایران را از طریقِ یک **آی‌پیِ تمیزِ Cloudflare** وصل می‌کنی.

> 🚀 در نسخه‌ی نهایی، این کلِ فرایند (دامنه، DNS، سرت، آی‌پیِ تمیز) **به‌صورتِ خودکار از داخلِ پنل** انجام می‌شود.

## 🧩 بعد از تونل، در پنلِ Xray (سنایی / 3x-ui) چه کنیم؟

تونل فقط **ترافیک را از مرز رد می‌کند**. کارِ اصلیِ پروکسی (مدیریتِ کاربر، ساختِ کانفیگ) را **Xray** انجام می‌دهد، که با پنلِ معروفِ **3x-ui (سنایی)** مدیریت می‌شود.

### ۱. نصبِ 3x-ui روی سرورِ خارج

```bash
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

### ۲. ساختِ یک inbound در 3x-ui

یک **Inbound جدید** بساز با این تنظیماتِ دقیق:

| فیلد | مقدار |
|---|---|
| Protocol | `VLESS` |
| Listen IP | `127.0.0.1` |
| Port | همان پورتی که `target` تونلت به آن اشاره می‌کند (مثلاً `2500`) |
| Security | **`none`** ← بدونِ TLS، بدونِ REALITY |
| Network | `tcp` |

> ⚠️ **مهم‌ترین نکته:** چون تونل + CDN خودشان لایه‌ی TLS/مخفی‌کاری را دارند، Xray را **ساده** بگذار (بدونِ TLS و REALITY). اگر روی Xray هم TLS بگذاری، دو لایه می‌شود و کند و دردسرساز است — و REALITY اصلاً از پشتِ CDN کار نمی‌کند.

### ۳. پورت را با تونل هماهنگ کن

پورتِ inbound در Xray باید **دقیقاً** همان پورتی باشد که در تونلِ سرور خارج به‌عنوان `-target` گذاشته‌ای:

```
t2tunnel server:  -target 127.0.0.1:2500
3x-ui inbound:    Port = 2500
```

اگر این دو فرق کنند، اتصال برقرار نمی‌شود.

### ۴. کانفیگِ کاربرِ نهایی را بساز

UUID را که 3x-ui ساخت کپی کن. حالا برای کاربر یک کانفیگِ VLESS بساز، اما با این تفاوتِ کلیدی:

| فیلد | مقدار |
|---|---|
| Address | **آی‌پیِ سرورِ ایران** (نه خارج!) |
| Port | پورتِ listenِ کلاینتِ ایران (مثلاً `9000`) |
| Protocol | `VLESS` |
| UUID | همان UUIDی که 3x-ui ساخت |
| Security | `none` |
| Network | `tcp` |

کاربر فکر می‌کند به یک آی‌پیِ تمیزِ ایرانی وصل است، ولی پشتِ صحنه ترافیکش از تونل رد می‌شود و از سرورِ خارج درمی‌آید. 🎯

### مسیرِ کاملِ ترافیک

```
v2rayNG کاربر  →  سرور ایران :9000 (تونل client)
              ══ تونل (TLS / WS پشتِ CDN) ══▶
              سرور خارج (تونل server)  →  Xray :2500 (3x-ui)
              →  اینترنتِ آزاد
```
